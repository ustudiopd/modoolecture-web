# 질문 일괄 삽입 템플릿

이 문서는 200개 이상의 질문을 데이터베이스에 일괄 삽입하기 위한 템플릿입니다.

## 사용 방법

1. 아래 형식에 맞춰 질문들을 작성합니다.
2. 이 MD 파일을 제공해주시면 자동으로 데이터베이스에 삽입하는 스크립트를 생성하겠습니다.

## 필수 정보

**이벤트 정보 (파일 상단에 한 번만 작성)**

```yaml
event_slug: "your-event-slug"  # 예: "2025-ai-seminar"
# 또는
event_id: "uuid-here"  # 이벤트 UUID를 알고 있다면
```

## 질문 형식

각 질문은 `---` 구분자로 분리하고, YAML 프론트매터와 본문으로 구성합니다.

### 기본 형식 (최소 필수 필드)

```markdown
---
title: 질문 제목 (선택, 없으면 content의 첫 100자 사용)
---

질문 내용을 여기에 작성합니다.
여러 줄로 작성 가능합니다.
```

### 전체 필드 예시

```markdown
---
title: AI 활용 막막한 행정직원: 실무 적용 및 가치 창출 가이드
primary_topic: workflow_automation
secondary_topics:
  - tools_models
  - prompting
intent: howto
display_name_raw: 홍길동
display_name_masked: 홍*동
category: Business
tags:
  - 자동화
  - 업무
is_answered: true
like_count: 5
---

데이터 접근 권한이 제한적인 행정직원의 입장에서, AI를 실무에 어떻게 적용해 업무를 자동화하고 가치를 창출할 수 있을까요?

## 답변 (선택)

답변 내용을 여기에 작성할 수 있습니다.
마크다운 형식으로 작성 가능합니다.

### Gemini 답변 (선택)

```markdown
---
answer_gemini: |
  Gemini로 생성된 답변 내용을 여기에 작성합니다.
  여러 줄로 작성 가능합니다.
---
```

### GPT 답변 (선택)

```markdown
---
answer_gpt: |
  GPT로 생성된 답변 내용을 여기에 작성합니다.
  여러 줄로 작성 가능합니다.
---
```

## 필드 설명

### 필수 필드
- **content**: 질문 내용 (본문에 작성)

### 선택 필드 (YAML 프론트매터)

| 필드 | 타입 | 설명 | 예시 |
|------|------|------|------|
| `title` | string | 질문 제목 | "RAG 구현 시 할루시네이션 잡는 법" |
| `primary_topic` | string | 주제 태그 (10개 중 1개) | `workflow_automation` |
| `secondary_topics` | array | 보조 주제 태그 (0~2개) | `["tools_models", "prompting"]` |
| `intent` | string | 의도 태그 (6개 중 1개) | `howto` |
| `display_name_raw` | string | 작성자 이름 (원본) | "홍길동" |
| `display_name_masked` | string | 작성자 이름 (마스킹) | "홍*동" |
| `category` | string | 카테고리 (레거시) | "Tech/Dev" |
| `tags` | array | 태그 배열 | `["RAG", "할루시네이션"]` |
| `is_answered` | boolean | 답변 완료 여부 | `true` |
| `like_count` | number | 좋아요 수 | `5` |
| `answer` | string | 일반 답변 (마크다운) | 본문에 작성 |
| `answer_gemini` | string | Gemini 답변 (마크다운) | 본문에 작성 |
| `answer_gpt` | string | GPT 답변 (마크다운) | 본문에 작성 |

### 토픽 태그 목록 (primary_topic, secondary_topics)

- `getting_started` - 입문/일반 활용/시작
- `workflow_automation` - 실무 적용/자동화/워크플로
- `prompting` - 프롬프트/질문법
- `tools_models` - 툴/모델 선택·비교·연동
- `accuracy_verification` - 정확도/환각/검증
- `security_privacy` - 보안/개인정보/사내정책
- `copyright_ethics` - 저작권/윤리/표절/출처
- `cost_roi` - 비용/구독/ROI
- `trends_learning_career` - 트렌드/학습/미래·커리어
- `none` - 무응답/의미없음 (일반적으로 사용 안 함)

### 의도 태그 목록 (intent)

- `howto` - 방법/가이드 요청
- `recommend` - 추천/비교/선택
- `troubleshoot` - 문제/불편/장애
- `explain` - 개념/원리 이해
- `strategy` - 전략/전망/의사결정
- `other` - 애매함

## 주의사항

1. **구분자**: 각 질문은 `---` (3개의 하이픈)으로 구분합니다.
2. **YAML 프론트매터**: 각 질문은 YAML 형식의 메타데이터로 시작합니다.
3. **본문**: YAML 프론트매터 다음에 질문 내용을 작성합니다.
4. **배열 필드**: `secondary_topics`, `tags`는 YAML 배열 형식으로 작성합니다.
5. **여러 줄 텍스트**: `answer`, `answer_gemini`, `answer_gpt`는 YAML의 `|` 또는 `>` 문법을 사용하거나 본문에 작성합니다.
6. **이벤트 정보**: 파일 상단에 `event_slug` 또는 `event_id`를 한 번만 작성합니다.

## 간단한 예시 (최소 필드만 사용)

```markdown
# 이벤트 정보
event_slug: "2025-ai-seminar"

---

---
---

질문 내용만 작성해도 됩니다. 제목은 자동으로 생성됩니다.

---

---
---

또 다른 질문입니다.
```

이 형식으로 작성해주시면 자동으로 데이터베이스에 삽입하는 스크립트를 생성하겠습니다!
